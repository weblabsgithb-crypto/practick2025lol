Цель

Создать прототип централизованной системы отчётности для Московского Политеха, включающий сбор данных из нескольких источников, унификацию метрик, интерактивную визуализацию и предикативную модель для прогнозирования операционных показателей (посещаемость, успеваемость, загрузка аудитории, инциденты)

Область применения

  • Управление расписанием и загрузкой аудиторий
  • Прогноз посещаемости и вовлечённости студентов
  • Мониторинг академической успеваемости и раннее обнаружение рисков
  • Поддержка принятия решений деканатами и учебными подразделениями

Техническая архитектура

  • Источники: CSV/Excel экспорты из учебных систем, журналы посещаемости, расписания.
  • ETL: скрипты на Python (pandas), нормализация и агрегация данных.
  • Хранилище: локальная база SQLite или JSON‑файлы для прототипа.
  • Аналитика: Jupyter Notebook для исследования данных и обучения модели.
  • Визуализация: фронтенд‑прототип (HTML/JS, графики Chart.js / Plotly) для дашбордов.
  • Модель прогнозирования: простая модель (линейная регрессия / RandomForest / SARIMA) с оценкой по RMSE/MAE.
